# Backend Visual Map Implementation Plan

Status: Complete (Phases 1-30 implemented; RDB productization phases 31-40 implemented)
Scale: Large
Owner decisions: complete through interview round 4
Implementation rule: phase order is mandatory; do not build future phases early.

## Goal

Build a Windows-first Tauri + React desktop app that bundles codebase-memory and rdb-memory engines, indexes a Git repository and one active RDB profile, and renders a visual 2-hop Backend Visual Map for backend developers.

## One-Line Product

Git repo와 RDB 스키마를 입력하면 API -> code -> DB 관계를 자동으로 시각화해주는 로컬 데스크탑 앱.

## Non-Goals

- No DB row data reads.
- No arbitrary SQL query console.
- No BI dashboard.
- No 3D graph visualization.
- No team cloud sync in v1.
- No PR/CI integration in v1.
- No automatic code modification.
- No automatic migration execution.
- No password storage in v1.
- No fork/merge of codebase-memory-mcp.

## First Product Scope

In scope:

- Tauri + React desktop shell.
- Bundled engine sidecars.
- Workspace storage under app-local data.
- Repo indexing through codebase-memory CLI.
- DB indexing through rdb-memory CLI.
- One active DB profile at a time.
- API/code/DB inventory panels.
- React Flow visual map canvas.
- Explore Mode.
- API Flow Mode.
- Table Usage Mode.
- Column Impact Mode.
- Search Mode.
- Candidate code-to-DB links with confidence and evidence.
- Inspector panel.
- Basic scan logs and errors.
- Windows installer/release package.

Deferred:

- Team workspace sharing.
- PR Impact Mode.
- Migration Risk Mode.
- Test Scope Mode.
- Architecture Drift Mode.
- OS credential store.
- Multi-DB simultaneous maps.
- Custom confirmation/training system for candidate links.

## Shared Contracts

### Workspace

```ts
type Workspace = {
  id: string;
  name: string;
  repoPath: string;
  codeProject?: string;
  dbProfiles: DbProfile[];
  activeDbProfileId?: string;
  createdAt: string;
  updatedAt: string;
};
```

### DB Profile

```ts
type DbProfile = {
  id: string;
  name: string;
  source: "sqlite" | "ddl-sqlite" | "postgres" | "mysql" | "sqlserver" | "oracle";
  path?: string;
  host?: string;
  port?: number;
  database?: string;
  username?: string;
  cachePath: string;
  lastIndexedAt?: string;
  passwordStored: false;
};
```

### Engine Result

```ts
type EngineRunResult = {
  ok: boolean;
  stdout: string;
  stderr: string;
  exitCode: number | null;
  startedAt: string;
  finishedAt: string;
};
```

### Visual Map

```ts
type VisualMap = {
  id: string;
  workspaceId: string;
  mode: "explore" | "api-flow" | "table-usage" | "column-impact" | "search";
  focus: EngineRef;
  nodes: VisualNode[];
  edges: VisualEdge[];
  warnings: string[];
};
```
