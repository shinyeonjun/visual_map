# Phase Reports

Every implementation phase for Backend Visual Map must create:

```text
docs/reports/backend-visual-map.phase-N.md
```

Each report should include:

- Phase number and title
- Status: Complete / Blocked / Failed
- Changed files
- What was implemented
- What was deliberately skipped
- Commands run
- Test/check results
- Known risks
- Next recommended phase

Operational troubleshooting and cross-phase incident notes are tracked in:

```text
docs/reports/backend-visual-map.troubleshooting-log.md
```
